# Changelog

All notable changes to this project should be documented in this file.

Use a simple structure based on Keep a Changelog.

Recommended sections:
- `Unreleased`
- version header such as `1.0.0 - YYYY-MM-DD`
- `Added`
- `Changed`
- `Fixed`

Guidance:
- Keep entries short and user-facing.
- Group technical cleanup under `Changed` or `Fixed` only when it affects users or contributors.
- Add a new version section when a release is tagged.
- Leave this file intentionally minimal until the first public release is cut.

Template:

## [Unreleased]

### Added

### Changed

### Fixed
